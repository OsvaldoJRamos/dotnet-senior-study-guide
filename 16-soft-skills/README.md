# 16 - Soft Skills

Senior engineering is not only code. Requirements, stakeholders, communication, and technology judgment determine whether a technically correct solution ever gets shipped or adopted. This section groups the non-code skills you're expected to demonstrate at a senior / lead level.

3. [Stakeholders](03-stakeholders.md) - Who they are, Mendelow matrix, balancing conflicting interests
4. [User Stories and BDD](04-user-stories-and-bdd.md) - INVEST (Bill Wake), Given/When/Then (Dan North)
5. [Requirements Management](05-requirements-management.md) - Traceability, baselines, change control, MoSCoW / RICE / WSJF
6. [Communication and Audience](06-communication-and-audience.md) - Writing for engineers vs product vs execs; adapting diagrams and docs
7. [Technology Selection](07-technology-selection.md) - Boring Technology (Dan McKinley), innovation tokens, lock-in, LTS
