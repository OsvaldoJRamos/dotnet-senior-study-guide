DECISIONS MADE DURING THE REORGANIZATION OF THE ".NET C# Studies" MATERIAL
================================================================

Date: 2026-04-09

Destination: D:\repos\v2 (Git repository with Markdown)

================================================================
1. GENERAL STRUCTURE
================================================================

DECISION: Organize into 8 thematic categories with numbered folders.
REASON: The original PDF had no clear organization — memory topics appeared
mixed with design patterns, and C# concepts appeared in the middle of
parallelism content. Grouping by theme makes it easier to find what you need.

DECISION: Each category has a README.md with an index and links to subtopics.
REASON: Makes navigation easier both on GitHub and locally. The reader always
knows where they are and how to go back.

DECISION: Navigation links (previous/next/back) at the bottom of each file.
REASON: Allows sequential reading like a book, without needing to go back to
the index every time.

================================================================
2. ORGANIZATION BY CATEGORIES
================================================================

The PDF topics were redistributed as follows:

01-fundamentos-csharp/
  - Unified: "C#, .NET, ASP.NET Core", "General C# Concepts" (namespace, CLR/IL),
    "Double x Decimal x Float", "Method/Class/Attribute Modifiers"
  - REASON: All of this is language and platform fundamentals. In the PDF they were
    scattered across non-consecutive pages.

02-colecoes-e-linq/
  - Unified: "IEnumerable vs IQueryable", "IEnumerable x List", "Yield in Loops"
  - REASON: They are all about collections and ways to iterate/query data.

03-memoria-e-performance/
  - Unified: "Memory Management" (Stack/Heap), "Memory Optimization",
    "Memory Leak in C#", and part of "Optimization Techniques"
  - REASON: All these topics deal with how .NET manages memory and how to
    optimize memory usage. In the PDF they were in separate sections with
    duplicate content (GC appeared in 2 different sections).

04-concorrencia-e-paralelismo/
  - Unified: "Parallelism", "Race Conditions", "Deadlocks", "Semaphore"
  - REASON: These are intrinsically related topics. Race conditions and deadlocks
    are parallelism problems, and SemaphoreSlim is a solution for them.

05-arquitetura-e-padroes/
  - Unified: "SOLID", "Design Patterns", "YAGNI", "KISS", "DRY",
    "Object Calisthenics", "Tell Don't Ask", "SAGA"
  - REASON: All are software design/architecture principles and patterns.

06-aspnet-core/
  - Unified: "DI, IoC, DIP and Service Locator", "Singleton x Scoped x Transient",
    "OAuth 2.0"
  - REASON: These are features specific to ASP.NET Core and its infrastructure.

07-acesso-a-dados/
  - Unified: "ORM, Micro ORM or Raw ADO", "Entity Framework", "Database"
  - REASON: All deal with how to access and manage persisted data.

08-devops/
  - Source: "CI/CD"
  - REASON: Separate category for infrastructure and delivery practices.

4. CORRECTED CONTENT
================================================================

- OAuth 2.0: The PDF stated that "OAuth 2.0 is an authorization framework that
  can be used to authenticate users." This is imprecise. OAuth 2.0 is
  strictly about AUTHORIZATION. For authentication, OpenID Connect is used
  (which is a layer on top of OAuth 2.0). Kept the original explanation but
  clarified the distinction.

- "Singleton x Scoped x Transient": The original text about Scoped said that
  "it cannot be transient" in a confusing way. Rewrote the explanation more
  clearly, keeping the correct technical content.
